Signature & verification

NHP-REG is sent inside a Noise handshake keyed with the agent's Step-1 private key. Completing that handshake is the signature — it cryptographically proves the sender holds the private key for pubKey, without ever transmitting the private key or a separate signature field.

On receipt the server verifies, in order:

  1. OTP is valid, unused, unexpired, and was issued for this usrId / devId.
  2. Key binding — pubKey equals the key committed in the NHP-OTP request and the static key proven in the Noise handshake (mismatch → rejected).
  3. Only then is the key registered and NHP-RAK returned.

A stolen OTP alone is useless: without the matching private key the handshake fails, and a different key fails the binding check.

NHP-RAK Server → Agent
