How registration works

Two protocol messages bind your device key to your identity.

NHP-OTP binds your device's public key to your identity and triggers an emailed code. NHP-REG then proves you hold the matching private key and completes registration — the server replies with NHP-RAK. Everything travels inside an encrypted Noise channel; the code itself is never sent back to the server in the clear.
