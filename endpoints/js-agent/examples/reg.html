Registration Steps

Step 1: Generate key pair

Create a device key pair locally in your browser. The private key never leaves your device.

Must match the target server's scheme (Step 2).

Step 2: Get verification code via NHP-OTP

Choose the target server and identify yourself. The server binds your public key to this identity and emails a one-time code over the NHP-OTP message.

Auto-discovers available servers on change
Used as both your user identity and OTP delivery address.
Leave blank to auto-generate

Step 3: Register Agent Key via NHP-REG

No code? Go back and send a new one.

API Usage Examples

// npm install @opennhp/agent
import { NHPAgent } from '@opennhp/agent';

// ── Step 1: generate key pair ─────────────────
const agent = new NHPAgent({
  cipherScheme: 'curve25519',  // or 'gmsm'
  transport:    'relay',
  relayUrl:     'https://relay.opennhp.org/relay',
});
await agent.init();

agent.addServer({ publicKey: '<server-public-key-base64>' });
agent.setIdentity({
  userId:   'alice@example.com',
  deviceId: 'my-laptop-01',
});

// ── Step 2: request OTP (NHP-OTP) ─────────────
const otpResult = await agent.requestOtp(
  'example',               // Auth Service Provider ID
  { email: 'alice@example.com' },
);
// Server emails alice@example.com a 6-digit code

// ── Step 3: register key (NHP-REG) ────────────
const regResult = await agent.registerPublicKey(
  'example',               // same ASP ID
  '123456',               // OTP from email
  { email: 'alice@example.com' },
);

if (regResult.success) {
  console.log('Registered! Public key:', agent.getPublicKey());
  // regResult.expiresAt — unix-seconds, or undefined if no TTL
} else {
  console.error(regResult.error, regResult.errorCode);
}

The private key never leaves the device. registerPublicKey() sends the NHP-REG packet inside a Noise handshake that cryptographically proves key possession — no separate signature field needed.

import { NHPAgent, OtpResult, RegisterResult } from '@opennhp/agent';

const agent = new NHPAgent({
  cipherScheme: 'curve25519',
  transport:    'relay',
  relayUrl:     'https://relay.opennhp.org/relay',
});
await agent.init();

agent.addServer({ publicKey: '<server-public-key-base64>' });
agent.setIdentity({ userId: 'alice@example.com', deviceId: 'my-laptop-01' });

// Step 2 — NHP-OTP
const otp: OtpResult = await agent.requestOtp('example');
if (!otp.success) throw new Error(otp.error);

// Step 3 — NHP-REG (otp from email)
const reg: RegisterResult = await agent.registerPublicKey(
  'example', '123456',
);
if (reg.success) {
  const pubKey: string = agent.getPublicKey();
  const expiresAt: number | undefined = reg.expiresAt;
}

Full type definitions are shipped with the package. OtpResult and RegisterResult are both exported from @opennhp/agent.

Step 2 — NHP-OTP (request code)
# The NHP-OTP packet is binary (Noise-encrypted UDP/relay).
# Use the JS SDK above, or any language binding.
# Example via the HTTP relay endpoint:

curl -X POST https://relay.opennhp.org/relay/<server-fingerprint> \
  -H "Content-Type: application/octet-stream" \
  --data-binary @nhp-otp-packet.bin

# Build nhp-otp-packet.bin with the CLI tool:
nhp-agentd otp \
  --server  <server-public-key-base64> \
  --user    alice@example.com \
  --device  my-laptop-01 \
  --asp     example \
  --relay   https://relay.opennhp.org/relay
Step 3 — NHP-REG (register key)
# After receiving the 6-digit code by email:
nhp-agentd register \
  --server  <server-public-key-base64> \
  --user    alice@example.com \
  --device  my-laptop-01 \
  --asp     example \
  --otp     123456 \
  --relay   https://relay.opennhp.org/relay

# On success the server responds with NHP-RAK:
# { "errCode": "0", "aspId": "example", "expiresAt": 1234567890 }

The CLI tool ships with OpenNHP (nhp-agentd). For Go applications use the endpoints/agent package directly; for other languages implement the Noise handshake and wrap the JSON payload per the NHP-OTP / NHP-REG message spec.
